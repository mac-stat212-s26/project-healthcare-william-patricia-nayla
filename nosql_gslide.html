Topic
Non-Relational (NoSQL) Databases: How they differ from SQL and how to use MongoDB in R with mongolite
Motivation
In class, we covered SQL databases in depth using DBI and duckdb. But SQL requires a fixed schema defined before inserting data — a constraint that breaks down for web-scraped data, API responses, and real-world data science workflows where fields vary per record. I wanted to understand what databases exist for these cases, how they differ structurally from SQL, and whether R has good tooling to work with them.

Findings

The unclear concept I researched was non-relational (NoSQL) databases — what they are, how they differ from SQL, and how to use them from R with the mongolite package. Unlike SQL, which enforces a fixed schema on every row, MongoDB stores data as flexible JSON documents where each record can have completely different fields. I demonstrated this using the ggplot2::diamonds dataset inserted into a live local MongoDB instance via three operations: m$insert() / m$find(), m$aggregate(), and m$update() / m$remove().

1. insert + count + find (A.1): m$insert(diamonds) loads all 53,940 rows into MongoDB in one call. m$find() with a JSON query string filters records and returns a standard R data frame — no SQL required. The query {"cut":"Premium","price":{"$lt":1000}} is equivalent to a SQL WHERE clause, and adding sort + limit mirrors ORDER BY ... LIMIT.

2. aggregate pipeline (B.2): m$aggregate() runs a computation pipeline inside MongoDB itself before returning results to R — equivalent to group_by(cut) |> summarize(avg_price = mean(price)) but executed on the server. This is a key advantage over SQL's GROUP BY: large aggregations run where the data lives, reducing what gets sent back to R.

3. update + remove + requery (C.3): m$update() uses $set to add a new field (flagged: true) to matching documents without touching other fields — something SQL's rigid schema makes cumbersome. m$remove() deletes records matching a filter. A subsequent m$find() shows the database state after both mutations, visualized as a boxplot.
